Match Expression which accepts a string, consuming no input, if its subexpression also accepts the string at the current position.
NotMatch Expression which accepts a string, consuming no input, only if its subexpression does not accept the string at the current position.
OneOrMore Expression which accepts a string if its subexpression matches the string one or more times in succession.
Optional Expression which always accepts the input string, optionally matching its subexpression against the string if possible.
OrderedChoice

Composite expression which represents a prioritised choice of multiple subexpressions, accepting a string if one of its subexpressions accepts the string.

Subexpressions are matched against the input string in the order they are added.
ParseExpression Abstract interface for a parsing expression.
PatternTerminal Expression which accepts a string if it matches a regular expression pattern.
RuleRef Expression which evaluates a named rule as a nested expression.
Sequence Composite expression which evaluates a sequence of nested expressions, accepting a string if it matches each expression in turn, and rejecting it otherwise.
